# Deephome Web —— 产品与技术规格 (v1)

Airbnb 风格的找房网页。用户可买/租搜索。差异化不在"关键词匹配",而在**理解意图 + 解释为什么推荐**。

---

## 1. 现实约束（必须先认清，它决定 v1 长什么样）

用真实 key 实测得到的硬事实：

| 能力 | 免费 Preview/sandbox 档 | 干净样例数据(14 套) |
|---|---|---|
| 房源量 | 真实全量(全省 12 万) | 固定 14 套多伦多 |
| 结构化字段(价/卧/卫/位/经纬度) | ✅ 干净 | ✅ 干净 |
| 文本(描述/地址/房型) | ❌ 字符打乱 | ✅ 干净 |
| 朝向/楼层 | ❌ 常缺失 | ✅ 有 |
| 照片 | ❌ 占位图 `sandbox/IMG-…` | 需配精选占位图 |
| 采光/景观 意图 | ❌ 无效(依赖上面几项) | ✅ 有效 |
| AI 图搜 / Places | ❌ 403 需付费 | — |
| realtor.ca 深链 | ❌ 响应里根本没有 | — |

**结论**：真实免费数据"能搜但难看"(乱码描述 + 占位图 + 意图失效)；样例数据"好看且意图有效"但非真实全量。
→ 网页做成**数据源可插拔**(引擎已是 `ListingSource`)。v1 默认走样例数据把体验和意图价值做足，真实 Repliers 作为可切换后端。**(见文末决策，待拍板)**

---

## 2. 架构

- **Next.js 全栈**(与 deepzeno 同栈，你熟)。
- **后端 API `/api/search`**：服务端运行 Deephome 意图引擎(直接复用现有 `src/` 模块) + 数据源。**Repliers key 只在服务端**，绝不进浏览器。
- **前端**：React + Tailwind，Airbnb 风(大留白、圆角卡片、克制排版、无花哨)。
- **图片**：真实照片 `https://cdn.repliers.io/{images[i]}`(CDN 支持动态裁剪)；样例用精选占位图。

---

## 3. 搜索交互设计（核心 —— 我的推荐）

两层输入，正好对应引擎的"硬约束 vs 软意图"：

1. **Header 筛选栏 = 硬约束**：买/租切换、地点、卧室数、卫浴数、价格区间。就是 Airbnb 那一行 filter。
2. **意图输入框 = 软意图(Deephome 的灵魂)**：自由文字，如"采光好、安静、能养狗"。提交后**回显引擎的理解**——"把『大窗户』理解为采光好，将按 朝向/楼层 排序"；歧义时弹一个澄清 chip(大窗户 = 采光? 景观? 空间感?)。
3. **结果卡片带"为什么"**：每张卡直接显示命中属性("朝南 · 28楼 · 采光好 ✓")。这是别家没有的——把引擎的 explanation 直接搬进 UI，让"可解释"成为卖点。

流程：设 filter + 打意图 → 搜索 → 意图排序结果，每张卡解释原因。

---

## 4. 房源展示

- **卡片**：封面图、价格、买/租标签、卧/卫、面积、小区、"为什么"chips、收藏心形。
- **详情**：点开 → **模态弹窗**(原页快速看) + 独立可分享页 `/listing/[mls]`(可新 tab 打开)。含图册、全部细节、地图。
- **"查看房源"按钮**：Repliers 不提供 realtor.ca 深链，故 v1 →
  - 主：打开 Deephome 自己的详情页(这本就是自建门户的意义)；
  - 次：best-effort 外链"去 realtor.ca 搜索此 MLS# / 地址"。
  - 真正的 realtor.ca 深链需付费档 / DDF 授权，届时再接。

---

## 5. 模块拆分（一个一个做，每个都能独立验收）

- **P0 · 脚手架 + 本 spec**（现在）：Next.js 骨架、目录结构、把引擎接进来的方式定好。
- **P1 · 搜索后端 `/api/search`**：包住引擎 + 数据源，先用 JSON/curl 验证正确，**完全不碰 UI**。
- **P2 · 结果展示层**：`ListingCard`(Airbnb 卡片) + 结果网格，先喂静态数据 → 再接 P1 的 API。
- **P3 · 搜索控件**：Header filter 栏 + 意图输入框，接 API，显示"理解到的意图"回显。
- **P4 · 房源详情**：模态 + `/listing/[mls]` 页 + 图册 + 地图 + 按钮。
- **P5 · 打磨**：loading / 空态 / 响应式 / 收藏 / 澄清 chip 交互。

每阶段做完、验收通过，再进下一个。不一口气堆完，避免 bug 堆积、难维护。
